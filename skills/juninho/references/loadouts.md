# Loadouts

Loadouts are reusable bundles. Juninho can equip them as-is or trim them.

## Starter Loadout

Use for vague product work.

- `brainstorming`
- `bias-check`
- `devils-advocate`

## Discovery Loadout

Use for user understanding and opportunity discovery.

- `interview`
- `user-interview`
- `jtbd-map`
- `user-needs-map`
- `assumption-test`
- `ost-builder`

## Strategy Loadout

Use for prioritization, strategy, and sequencing.

- `gist-plan`
- `ice-score`
- `wardley-map`
- `cynefin-classify`
- `devils-advocate`

## Spec Loadout

Use for turning product decisions into Linear artifacts.

- `linear-spec`
- `linear-issues`
- `service-check`
- `usability-check`

## Launch Loadout

Use for go-to-market and readiness decisions.

- `launch-tier`
- `metrics-detect`
- `service-check`
- `retrospective`

## Rule

Equip a named loadout only when most skills are relevant. Otherwise select individual skills.
