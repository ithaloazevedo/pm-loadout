# Curation

## Included

| Skill | Why it belongs |
|---|---|
| `juninho` | Orchestrates PM workflows and specialist agents. |
| `brainstorming` | Turns vague ideas into clearer product direction. |
| `linear-spec` | Structures initiatives, discovery projects, delivery projects, and PM issues. |
| `linear-issues` | Shapes PM work into Linear issues. |
| `bias-check` | Reduces decision and research bias. |
| `devils-advocate` | Challenges premature certainty and weak narratives. |
| `interview` | Onboards product/project context. |
| `user-interview` | Guides story-based user interviews. |
| `jtbd-map` | Maps jobs to be done. |
| `user-needs-map` | Separates user needs from solutions. |
| `assumption-test` | Designs small tests for risky assumptions. |
| `ost-builder` | Builds Opportunity Solution Trees from evidence. |
| `gist-plan` | Connects goals, ideas, steps, and tasks. |
| `ice-score` | Prioritizes opportunities with evidence-backed confidence. |
| `wardley-map` | Maps value chain and strategic context. |
| `cynefin-classify` | Chooses methods based on problem domain. |
| `service-check` | Checks service-level product quality. |
| `usability-check` | Checks interface-level usability. |
| `launch-tier` | Classifies launch size and readiness. |
| `metrics-detect` | Identifies relevant product metric sources. |
| `retrospective` | Captures learning after a product cycle. |

## Excluded by design

Engineering-centric skills stay out of this repo unless a PM-specific wrapper is added:

- code review;
- automated testing;
- CI debugging;
- git worktrees;
- branch finishing;
- implementation bootstrapping;
- system debugging;
- programming TDD.

## Optional future additions

- `a11y-check` for PMs responsible for accessibility.
- `privacy-check` for regulated or data-heavy products.
- `regulatory-review` for health, finance, public sector, education, and AI products.
- `team-shape` for org design and team topology decisions.
